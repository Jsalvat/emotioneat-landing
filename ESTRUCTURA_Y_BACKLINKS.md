# Estructura de Páginas y Backlinks - EmotionEat Landing

## Estructura General de la Landing Page

### Flujo de Secciones (index.astro)

```
1. Header (Fixed)
   └─ Logo + Mega Menu + CTA Button

2. Hero Section
   ├─ Pre-headline: "You're not broken. You're human."
   ├─ Headline: "What if eating wasn't a battle anymore?"
   ├─ Pain point description
   ├─ Hope line
   ├─ 2 CTAs (Primary + Secondary)
   └─ Trust signals

3. Empathy Section (NUEVA)
   ├─ Título: "If this sounds familiar, you're in the right place"
   ├─ 4 Pain Points Cards
   └─ Mensaje de cierre empático

4. How It Works
   ├─ 3 pasos humanizados
   └─ Líneas conectoras visuales

5. Features
   ├─ 2 features grandes
   └─ 3 features medianas

6. Testimonials (NUEVA)
   └─ 3 testimonios con quotes

7. FAQ
   ├─ 7 preguntas (6 originales + 1 nueva empática)
   └─ Contact CTA

8. CTA Final
   ├─ Título empático
   ├─ CTA principal
   └─ 3 trust indicators emocionales

9. Footer
   ├─ 4 columnas
   └─ Bottom bar
```

## Notas de Implementación

### SEO Considerations
- Todos los 15 enlaces SEO están presentes tanto en Header como en Footer (excepto 1)
- Los enlaces están organizados por categorías temáticas
- Cada enlace mantiene su URL exacta sin cambios
- Los anchor links (#) permiten navegación interna fluida

### Accesibilidad
- Mega-menu con `aria-expanded` y `aria-haspopup`
- Mobile menu con `aria-label` en el botón
- Todos los enlaces tienen hover states claros
- Focus states con `focus:ring` para navegación por teclado

### Performance
- Mega-menu se carga con la página (no lazy)
- Transiciones CSS puras (no JavaScript)
- Mobile menu con `overflow-y-auto` para listas largas
- Cierre automático del mobile menu al hacer clic en anchors

## Archivos Modificados

### Header
- `src/components/Header.astro` - Contiene el mega-menu y mobile navigation

### Footer
- `src/components/Footer.astro` - Contiene las 4 columnas con enlaces organizados

### Lógica de Enlaces
Los enlaces están definidos como objetos en ambos componentes:

```javascript
const resourceLinks = {
  emotionalEating: [ /* 6 enlaces */ ],
  stressAndBoredom: [ /* 5 enlaces */ ],
  understandingTriggers: [ /* 4 enlaces */ ]
};
```

Esta estructura permite:
1. Fácil actualización de enlaces
2. Consistencia entre Header y Footer
3. Mantenimiento centralizado
4. Fácil añadir/eliminar enlaces en el futuro
